# AÏKO Demo Commands

Use these copy-paste prompts during an MVP demo. They are intended to show planning, supervision, approvals, files, and controlled self-improvement without sending real external messages.

## CEO Introduction

```text
Hello, what are you?
```

Expected: AÏKO explains that it is an AI Marketing Operating System with a CEO, agents, Web Operators, approvals, reports, files, and controlled self-improvement.

## Create Demo Project

```text
Create a marketing project for Demo Parking.
```

Expected: AÏKO creates or opens a Demo Parking project, records project context, and shows project-related chips or links.

## Project Recall

```text
What are we doing for Demo Parking?
```

Expected: AÏKO answers from project memory and summarizes the current objective, strategy, tasks, or next recommended action.

## Executive Report

```text
Generate an executive report for Demo Parking.
```

Expected: AÏKO creates an internal report artifact. Open the Reports tab or `/files` to inspect/export Markdown or JSON.

## Web Operator Draft Task

```text
Kevin, open Canva and create a draft Instagram post for Demo Parking.
```

Expected: AÏKO detects the Canva skill/playbook, opens Canva directly through the Web Operator, pauses for login/security if needed, and does not publish/share/download without approval.

## Missing Capability Strategy

```text
For Demo Parking, the best strategy is WhatsApp outreach. Can AÏKO execute this?
```

Expected: AÏKO creates an internal strategy execution plan, detects missing capability such as `whatsapp_web` / `whatsapp_outreach`, creates a System Improvement Proposal and Codex-ready prompt, and does not open WhatsApp or send messages.

## Self-Improvement Status

```text
What is the status of AÏKO self-improvement?
```

Expected: AÏKO summarizes current proposals, lifecycle statuses, missing capabilities, approved/in-progress/pending validation work, and validated capabilities from `/system`.

## Safety Reminder

For any prompt involving external-facing actions, confirm that AÏKO routes risky actions through approvals and does not silently execute sends, posts, messages, publishing, sharing, or downloads.
